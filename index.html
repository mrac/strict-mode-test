Strict Mode test

// All features test - if the web browser supports ECMAScript 5 "Strict Mode" - evaluation, semantics and exceptions.

Inpiration

The test was inspired by the famous strict-mode test coded by Kangax and of course ECMAScript 5 specification (PDF).

Browser support

As for today (April 1, 2011) the strict mode is fully supported only by FireFox 4 web browser.

Why to "use strict"?

Strict mode environment is going to be much better world for js developers.

Incredible possilities of ECMAScript 5 of sealing and freezing object properties together with Strict Mode exceptions bring consistency and predictibility, that are very needed in complex, big-scale projects.

Use with caution

But beware, use it with caution because the strict mode will not only rise exceptions but also will change the way your code is evaluated. We need to be sure your code behave the same way in strict mode and in unstricted environment. Listen what Brendan Eich is saying about that:

ES5 Strict Mode, Use With Caution (by Brendan Eich)

Usage example

			
				var arr = [];
				
				// strict scope
				(function() {
					"use strict";
					
					delete arr.length; // error thrown!
					
				})();
				
				// non-strict scope
				(function() {
				
					delete arr.length; // silently fails
					
				})();
