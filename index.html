Strict Mode test

// Testing whether given web browser supports ECMAScript 5 "Strict Mode" (evaluation, semantics and exceptions).

Inspiration

The test was inspired by other strict-mode test coded by Kangax and of course by studying ECMAScript 5 specification.

Browser support

As for today (January 25, 2013) considering browsers on desktop computers:

  • The strict mode is fully supported by FireFox 18, IE 10, Safari 6 and Opera 12.
  • Chrome 24 still has one issue that does not conform with the strict-mode spec.

Why to "use strict"?

Strict mode environment is going to be much better world for js developers.

ECMAScript 5 possibilities of sealing and freezing object properties together with Strict Mode exceptions bring consistency and predictibility, that are very needed in complex, big-scale projects.

Use with caution

The strict mode will not only rise exceptions but also will change the way JavaScript code is evaluated. So we need to be sure our code behave the same way in strict mode and in unstricted environment. Listen what Brendan Eich is saying about that:
ES5 Strict Mode, Use With Caution (by Brendan Eich)

Usage example


var arr = [];

// strict scope
(function() {
	"use strict";
	
	delete arr.length; // error thrown!
	
})();

// non-strict scope
(function() {

	delete arr.length; // silently fails
	
})();
