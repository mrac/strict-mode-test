Strict Mode test

// Testing whether given web browser supports ECMAScript 5 "Strict Mode" (evaluation, semantics and exceptions).

Use with caution

How we can "use strict" to be backward-compatible with old browsers that don't support strict mode?
There are only 3 rules:
  • Make sure that throwing errors is just a development help and not a part of application logic
  • Don't rely on the arguments object as separated from parameter variables
  • Don't rely on the default function context this as undefined

Usage example


var arr = [];

// strict scope
(function() {
	"use strict";
	
	delete arr.length; // error thrown!
	
})();

// non-strict scope
(function() {

	delete arr.length; // silently fails
	
})();
